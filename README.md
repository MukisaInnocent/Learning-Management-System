# Learning Management System (edu-platform)

A multi-tenant Learning Management System built as a Turborepo monorepo.

- **`apps/api`** — NestJS + Prisma + PostgreSQL REST API (JWT auth, role-based access)
- **`apps/web`** — Next.js (App Router) frontend with role-based dashboards (admin, teacher, student, parent)
- **`packages/shared-types`** — Types shared between the API and web apps
- **`docs/`** — Project planning and design notes

## Tech stack

| Layer    | Technology                                            |
| -------- | ----------------------------------------------------- |
| API      | NestJS 11, Prisma 6, PostgreSQL, Passport JWT, bcrypt |
| Web      | Next.js 16, React 19, Tailwind CSS 4, TanStack Query  |
| Tooling  | Turborepo, TypeScript, ESLint, Prettier               |

## Prerequisites

- Node.js 20+ and npm 10+
- A PostgreSQL database

## Local development

```bash
# 1. Install dependencies (from the repo root)
npm install

# 2. Configure environment variables
cp apps/api/.env.example apps/api/.env
cp apps/web/.env.example apps/web/.env.local
#   then edit each file with your values (DATABASE_URL, JWT_SECRET, NEXT_PUBLIC_API_URL)

# 3. Set up the database (run from apps/api)
cd apps/api
node_modules/.bin/prisma migrate dev
cd ../..

# 4. Run everything
npm run dev
```

The API runs at `http://localhost:4000/api/v1` and the web app at `http://localhost:3000`.

## Environment variables

**API (`apps/api/.env`)**

| Variable       | Example                                              |
| -------------- | ---------------------------------------------------- |
| `DATABASE_URL` | `postgresql://user:pass@127.0.0.1:5432/edu_platform` |
| `JWT_SECRET`   | a long random string                                 |
| `PORT`         | `4000`                                               |
| `FRONTEND_URL` | `http://localhost:3000` (used for CORS)              |

**Web (`apps/web/.env.local`)**

| Variable              | Example                          |
| --------------------- | -------------------------------- |
| `NEXT_PUBLIC_API_URL` | `http://localhost:4000/api/v1`   |

> `.env` files are git-ignored — never commit real secrets. Only `.env.example` files are tracked.

## Deployment (Render)

This repo ships a [`render.yaml`](./render.yaml) Blueprint that provisions:

- a managed **PostgreSQL** database (`edu-db`)
- the **API** web service (`edu-api`) — runs `prisma migrate deploy` on boot
- the **Web** service (`edu-web`)

**Deploy order matters** — `NEXT_PUBLIC_API_URL` is baked into the web bundle at build time, so the API must be live before the web builds.

1. In the Render dashboard choose **New + → Blueprint**, connect this GitHub repository, and apply.
2. Wait for `edu-db` and `edu-api` to finish deploying. Note the API's URL (e.g. `https://edu-api.onrender.com`).
3. In the `edu-web` service's **Environment** tab, set:
   - `NEXT_PUBLIC_API_URL` → `https://edu-api.onrender.com/api/v1`
4. In the `edu-api` service's **Environment** tab, set:
   - `FRONTEND_URL` → `https://edu-web.onrender.com`
5. Trigger a **Manual Deploy** on both services to pick up the new values.

> The `JWT_SECRET` is auto-generated by the Blueprint — no action needed.
